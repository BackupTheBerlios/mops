Спецификация формата пакета MOPSLinux

Версия 1.0 от 25.01.2007


Общий вид

  1. Пакет представляет собой архив формата tar.gz, сформированный таким образом, что будучи распакованным в корень файловой системы, все файлы устанавливаемой программы окажутся на своих местах.

  2. Кроме файлов, в корне архива находится папка /install, содержащая служебную информацию, необходимую для работы с пакетом: установочные скрипты и техническое описание.

  3. Имя файла, строго говоря, может быть произвольным, но для обеспечения совместимости с системой пакетов Slackware имя должно составляться по тем же правилам: имя-версия-архитектура-билд.tgz. Пример: perl-3.3.8-i486-2.tgz

Формат файла data.xml

Файл представляет собой XML-дерево, в соответствующих полях которого записаны необходимые данные.

Первая строка файла – XML-заголовок:

<?xml version="1.0" encoding="utf-8"?>

В нем говорится, что файл имеет формат XML версии 1.0 и записан в кодировке utf-8 (то есть в юникоде).

Далее идут непосредственно данные. XML-формат хранит данные в виде дерева. Элементы дерева будем называть нодами. Ноды представляют собой теги, которые могут иметь параметры. Например:

<name> имя_пакета </name> Нода без параметров

<short_description lang="ru"> короткое_описание </short_description> Нода с параметром lang, равным ru

Обязательными параметрами в данном файле являются имя, версия, архитектура и билд. Без них файл считается некорректным и обрабатываться не будет. Все остальные параметры можно опустить, если в них нет необходимости.

Описательные параметры (description, short_description, changelog) могут иметь параметр lang, обозначающий язык, на котором написано данное поле. В случае отсутствия данного параметра язык принимается равным английскому. Параметру можно присваивать значения в виде двухбуквенных кодов языка (ru, en, fr, de...)

Требования к версии задаются ключевыми словами. Проще было бы обозначать их общепринятыми математическими знаками (>=, <=, !=, ==, >, <...), но они пересекаются с форматом XML, и это очень усложнит ручное написание файла. Поэтому были введены ключевые слова (keywords), которые заменяют эти знаки:

Ключевое слово

Знак

Описание

more

>

Требуется версия больше чем указанная

less

<

Требуется версия меньше чем указанная

equal

=

Требуется точно указанная версия

notequal

!=

Требуется любая версия кроме указанной

atleast

>=

Требуется версия не менее указанной

notmore

<=

Требуется версия не более указанной

any

(любой)

Любая версия подходит.

(Данный вид соотношения можно не использовать: достаточно просто указать только имя)



В dependencies и suggests необязательно всегда указывать версию: если версия значения не имеет, ее можно просто опустить, указав только имя.

Также, можно комбинировать требования к версии: в таком случае надо указать несколько зависимостей по одному пакету с разными данными. Например, нам требуется QT версии не ниже 4.0, но только не 4.1. Тогда мы указываем две зависимости: QT >=4.0 и QT!=4.1





Пример файла (на базе описания пакета samba-3.0.23c-i486-1.tgz):

<?xml version="1.0" encoding="utf-8"?>
<package>
	<name>
		samba
	</name>
	<version>
		3.0.23c
	</version>
	<arch>
		i486
	</arch>
	<build>
		1
	</build>
	<short_description lang="en">
		 samba (SMB file and print server)
	</short_description>
	<description lang="en">
		Samba is an SMB file and print server for SMB and CIFS clients.  It
		allows you to make file space or printers on a Samba host available to
 		any PCs running SMB clients (such as PCs running Windows).

 		If you have any SMB servers (such as Windows NT/2K Server), you may be
 		able to replace them by or supplement them with Samba. One of
		Samba&apos;&apos;s big strengths is integration, so you can use it to tie
		together your Linux hosts and Windows PC clients.
	</description>
	<changelog lang="en">
		12.01.2007
			Bugfixes
	</changelog>
	<tags>
		<tag>
			windows
		</tag>
		<tag>
			server
		</tag>
	</tags>
	<maintainer>
		<name>
			AiX
		</name>
		<email>
			i27249@gmail.com
		</email>
	</maintainer>
	<configfiles>
		<conffile>
			/etc/rc.d/rc.samba
		</conffile>
		<conffile>
			/etc/samba/smb.conf
		</conffile>
	</configfiles>
	<dependencies>
		<dep>
			<name>perl</name>
		</dep>
		<dep>
			<name>curl</name>
			<condition>atleast</condition>
			<version>1.0</version>
		</dep>
	</dependencies>
	<suggests>
		<suggest>
			<name>bash</name>
		</suggest>
		<suggest>
			<name>kernel</name>
			<condition>atleast</condition>
			<version>2.6</condition>
		</suggest>
	<suggests>
</package>

Заключение, так сказать

Формат файла разрабатывался с целью обеспечения максимально возможной совместимости со старым форматом (Slackware pkgtools), при этом расширяя функциональность, и делая описание данных более четким. При наличии корректно составленного data.xml и скриптов (если они нужны), пакет можно собрать вызвав утилиту buildpkg без параметров из корневой директории с пакетом. Имя будет сгенерировано автоматически.
